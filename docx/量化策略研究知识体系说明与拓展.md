# 量化策略研究知识体系说明与拓展

> 本文档基于《量化策略研究知识体系概览》思维导图整理，对图中涉及的术语进行归纳、说明与拓展，供策略研究与实盘参考。

---

## 一、策略收益类（核心逻辑 / 收益风险）

本类关注策略的收益来源、风险度量及常见收益/风险情景，是评估与设计策略的基础。

### 1.1 收益分解：Alpha 与 Beta

| 概念 | 说明 | 拓展 |
|------|------|------|
| **Alpha（阿尔法）** | 相对基准的超额收益，即“跑赢大盘”的部分。 | 量化上常写为 \( R_p - R_f = \alpha + \beta(R_m - R_f) + \varepsilon \)。Alpha 是回归截距，代表与市场无关的收益。追求稳定正 Alpha 是指增、中性等策略的核心目标。 |
| **Beta（贝塔）** | 组合收益随市场（基准）涨跌的程度，即系统性/市场风险暴露。 | \(\beta = \frac{\mathrm{Cov}(R_p, R_m)}{\mathrm{Var}(R_m)}\)。Beta=1 表示与市场同幅波动；Beta>1 更激进，Beta<1 更保守。市场中性策略目标是将 Beta 压近 0。 |

### 1.2 策略形态：指增与中性

| 概念 | 说明 | 拓展 |
|------|------|------|
| **指增（指数增强）** | 在跟踪某一指数（如沪深300、中证500）的同时，通过选股/择时获取 Alpha。 | 约束通常包括：行业/风格相对基准的偏离、跟踪误差、换手率等。核心是在“跟踪误差预算”内最大化预期 Alpha。 |
| **中性（市场中性）** | 用股指期货、融券等工具对冲掉 Beta，只保留 Alpha，收益与大盘涨跌弱相关。 | 多空对冲：多头股票组合 + 空头股指期货/ETF/融券。需注意基差、对冲成本、风格漂移（如市值、行业敞口未完全中性）。 |

### 1.3 风险与性价比指标

| 概念 | 说明 | 拓展 |
|------|------|------|
| **夏普比率（Sharpe Ratio）** | 单位风险（波动率）对应的超额收益：\(\frac{R_p - R_f}{\sigma_p}\)。 | 假设收益近似正态时，夏普可衡量“风险调整后收益”。年化常用 \(\sqrt{252}\) 或 \(\sqrt{12}\) 乘日/月夏普。通常 >1 认为较好，>2 为优秀，但需结合样本期与策略类型。 |
| **卡玛比率（Calmar Ratio）** | 收益与最大回撤之比：年化收益 / 最大回撤。 | 更贴近“能承受多大回撤换多少收益”，适合风控和产品设计。与夏普互补：夏普看波动，卡玛看最差回撤。 |
| **最大回撤 / 回撤** | 净值从历史高点到之后最低点的最大跌幅。 | 回撤 = (峰值 - 谷值) / 峰值。影响客户体验和清盘风险，是产品与风控的核心约束之一。 |
| **清盘线** | 产品约定的止损线，净值跌破则强制清盘（常见如 0.85、0.90）。 | 设置需兼顾客户承受力与策略波动特性；过高会频繁触发，过低则亏损过大。常与最大回撤、波动率一起做压力测试。 |

### 1.4 回测与稳健性

| 概念 | 说明 | 拓展 |
|------|------|------|
| **过拟合 / 过度优化** | 策略过度贴合历史数据，参数或规则过于“量身定制”，样本外或实盘失效。 | 应对：简化模型、正则化、减少参数、留出样本外区间、滚动回测、蒙特卡洛参数扰动、实盘前小资金验证。 |
| **样本内 / 样本外** | 样本内：用于拟合/优化策略的数据；样本外：未参与拟合、用于验证的数据。 | 严谨做法：训练集（样本内）→ 验证集（调参）→ 测试集（样本外，仅做一次评估）。时间序列中需严格按时间划分，避免未来数据泄露。 |

### 1.5 策略状态与风险情景

| 概念 | 说明 | 拓展 |
|------|------|------|
| **薅羊毛** | 利用规则漏洞、短期价差或低风险套利机会获取小利。 | 如打新、可转债套利、部分事件套利。收益有限、容量有限，需关注规则变化与竞争加剧。 |
| **躺赢** | 策略表现稳定，净值曲线平滑上行，无需频繁人工干预。 | 多为中低频、逻辑清晰、风控到位的策略；依赖市场环境，需持续监控是否“卷”或失效。 |
| **炸仓** | 极端行情下亏损急剧放大，净值暴跌甚至爆仓。 | 常与高杠杆、未对冲、流动性枯竭、相关性骤升有关。风控需设置硬性止损、杠杆上限和压力情景。 |
| **裸奔** | 未做对冲，组合完全暴露于市场（或某一维度）风险。 | 指增多头、单边做多/做空且无对冲，均属“裸奔”。需明确产品定位：要 Beta 还是要纯 Alpha，并相应控制敞口。 |

## 二、因子数据类（策略底层 / 数据处理）

因子是收益预测与组合构建的输入；数据处理质量直接决定回测可信度与实盘表现。

### 2.1 因子概念与分类

| 概念 | 说明 | 拓展 |
|------|------|------|
| **因子** | 驱动股价或收益的可量化指标，用于预测收益、风险或构建组合。 | 常见分类：**量价**（动量、波动、流动性）、**基本面**（估值、盈利、成长、质量）、**情绪**（舆情、资金流、分析师）、**另类**（卫星、产业链、消费等）。多因子模型将多个因子加权或回归得到预期收益。 |
| **因子暴露** | 组合或个股在某一因子上的敏感度/权重。 | 若因子已标准化，暴露可理解为“组合在该因子上的得分”。控制暴露即控制风格（如市值、行业、Beta），避免风格漂移。 |

### 2.2 因子预处理与构造

| 概念 | 说明 | 拓展 |
|------|------|------|
| **因子中性化** | 剔除行业、市值等与“因子逻辑”无关的干扰，保留因子本身的选股信息。 | 常用方法：对因子做行业/市值等分组回归，取残差作为中性化因子；或约束组合在这些维度上的暴露为 0。 |
| **去极值 / 标准化** | 去极值：减弱极端值影响（如截尾、MAD、百分位）；标准化：统一量纲与分布（如 Z-Score）。 | 避免少数极端股票主导因子收益；标准化便于多因子合成与回归。需在截面、时间维度上一致处理，防止前视。 |
| **正交化** | 消除因子间共线性，使各因子尽量独立。 | 如对因子 A 做对因子 B 的回归取残差，得到与 B 正交的 A。可减少多重共线性、避免重复暴露、便于解释单因子贡献。 |

### 2.3 因子迭代与衰减

| 概念 | 说明 | 拓展 |
|------|------|------|
| **因子衰减** | 因子预测力或收益随时间减弱的现象。 | 原因：被广泛使用、市场结构变化、制度变化等。应对：持续挖掘新因子、缩短因子有效期、动态权重、结合多源信息。 |
| **因子挖掘 / 迭代** | 从数据与逻辑出发，发现新因子或改进已有因子。 | 流程：逻辑假设 → 指标构建 → 回测与 IC/IR 评估 → 组合内检验 → 实盘跟踪。需警惕过拟合与数据挖掘偏差。 |

### 2.4 回测偏差与数据质量

| 概念 | 说明 | 拓展 |
|------|------|------|
| **前视偏差（Look-ahead Bias）** | 回测中使用了当时尚未公开或尚未发生的信息。 | 例如：用了当日收盘价做当日开盘交易、用了未发布财报、用了未来才有的指数成分。会导致回测收益虚高，实盘无法复现。 |
| **幸存者偏差（Survivorship Bias）** | 回测仅使用当前仍上市、仍存续的标的，忽略已退市、长期停牌等。 | 会高估历史收益与策略有效性。严谨回测应使用“历史成分/历史可交易池”，或对退市、ST 等做专门处理。 |
| **Tick 级 / 分钟级** | 数据时间粒度：Tick 为逐笔成交，分钟为 1 分钟 K 线等。 | Tick 精度最高，用于高频、微观结构研究；分钟/日频用于中低频。粒度越细，数据量与计算量越大，需与策略频率匹配。 |
| **复权** | 对除权除息导致的价格缺口做调整，使历史价格可比。 | **前复权**：以当前价为基准向前调整，便于看当前价与历史价关系；**后复权**：以历史价为基准向后调整，便于看长期真实涨跌。回测需统一复权方式，避免价格断裂。 |
| **另类数据** | 非传统行情、财报的数据源。 | 如：舆情、新闻、卫星（停车量、夜光）、产业链、电商/消费、招聘等。可提供差异化信息，但存在噪音、延迟、可得性与合规问题。 |

## 三、交易执行类（实盘交易 / 订单操作）

执行层决定“信号”如何变成真实成交，成本与冲击直接影响实盘与回测的差异。

### 3.1 执行成本

| 概念 | 说明 | 拓展 |
|------|------|------|
| **滑点** | 下单价格（或预期价格）与实际成交价格的差异。 | 由行情变化、流动性、订单类型与规模导致。回测中常设固定比例（如 0.1%）或按订单规模建模；实盘需统计真实滑点并反馈到策略与成本假设。 |
| **冲击成本** | 大额订单自身推动价格朝不利方向变动，带来的额外成本。 | 与订单规模、流动性、下单速度相关。可粗略用平方根模型：冲击 ∝ \(\sqrt{\frac{Q}{V}}\)（Q 为成交量占比，V 为市场成交量）。 |

### 3.2 订单类型与算法

| 概念 | 说明 | 拓展 |
|------|------|------|
| **TWAP / VWAP** | **TWAP**：在一段时间内按时间均匀拆分订单；**VWAP**：按历史或预期成交量分布拆分，使成交曲线尽量贴合成交量曲线。 | 目标都是降低冲击、避免单笔过大。VWAP 更贴近市场成交节奏，但对成交量预测敏感。 |
| **冰山单 / 隐藏单** | 将大单拆小，仅展示部分挂单量，隐藏真实意图。 | 减少对市场的信号暴露，降低被逆向选择的概率；在流动性较差或大单时常用。 |
| **市价单 / 限价单** | **市价单**：以当前最优价尽快成交，成交优先、价格不确定；**限价单**：指定价格或更优价成交，价格可控、可能不成交。 | 大单、急单多用市价或算法单；可等待的建仓/减仓可用限价单降低成本。 |
| **条件单 / 止损止盈单** | **条件单**：满足设定条件（价格、时间等）后自动下单；**止损/止盈**：价格触及预设价位时自动平仓。 | 用于纪律性风控和自动化执行，需注意流动性差时的滑点与无法成交风险。 |

### 3.3 交易模式与仓位状态

| 概念 | 说明 | 拓展 |
|------|------|------|
| **做市** | 同时挂出买卖双向报价，提供流动性，赚取价差与手续费返还。 | 对速度、库存管理、风险控制要求高，多为专业机构与做市商。 |
| **骑手续费 / 薅手续费** | 通过高频或做市达到交易所/券商设定的量级，获取手续费返还。 | 收益依赖交易量与费率政策，需与策略收益、冲击成本一起核算是否划算。 |
| **日内回转 / T+0** | 同一交易日内完成开仓与平仓，利用日内波动获利。 | A 股股票实行 T+1，日内回转通常需底仓（先有持仓再当日卖出后买回）或使用融券、ETF、期权等 T+0 工具。 |
| **隔夜仓 / 裸空** | **隔夜仓**：持仓过夜，承担隔夜风险；**裸空**：无对应多头或现货的纯做空。 | 裸空风险大，受监管与融券约束；隔夜仓需考虑隔夜跳空与资金成本。 |
| **平仓 / 止盈 / 止损** | **平仓**：了结持仓；**止盈**：达到目标收益后离场；**止损**：达到可接受亏损后离场。 | 止盈止损是风控与资金管理的一部分，需事先设定规则并执行，避免情绪化。 |

## 四、技术工程类（系统 / 技术 / 开发）

技术架构支撑策略从研究、回测到实盘的全流程，尤其对高频与大规模因子计算至关重要。

### 4.1 延迟与基础设施

| 概念 | 说明 | 拓展 |
|------|------|------|
| **低延迟** | 从行情接收到订单抵达交易所的时间极短，是高频策略的核心要求。 | 涉及：网络、主机、操作系统、应用逻辑、交易所接口等各环节优化。微秒级、亚微秒级常见于做市与套利。 |
| **托管 / Colo** | 将服务器部署在交易所或金融数据中心机房，缩短物理距离与网络跳数。 | 降低延迟与抖动，是高频与对延迟敏感策略的常见选择；成本与运维要求较高。 |
| **裸光纤** | 点对点专用光纤，不经公网与多余设备。 | 延迟与稳定性优于普通专线，多为高频机构使用。 |

### 4.2 系统与开发

| 概念 | 说明 | 拓展 |
|------|------|------|
| **回测框架 / 交易引擎** | **回测框架**：在历史数据上模拟撮合与持仓，评估策略表现；**交易引擎**：实盘中接收信号、生成订单、管理状态、对接券商。 | 回测需尽量模拟真实滑点、冲击、涨跌停、停牌等；交易引擎需稳定、可监控、可回放与审计。 |
| **多线程 / 协程** | 利用多核与异步 I/O，提高数据处理与订单吞吐。 | 多线程适合 CPU 密集（如因子计算）；协程/异步适合 I/O 密集（行情、下单、数据库）。需注意线程安全与锁。 |
| **API 对接** | 通过券商或第三方提供的 API 获取行情、账户、下单。 | 需了解接口限制（频率、字段、稳定性）、认证与合规要求，并做好异常与重连处理。 |
| **量化平台** | 聚宽、米筐、QuantConnect 等提供数据、回测、模拟与实盘的一体化环境。 | 适合快速验证想法与中小规模实盘；大规模或极低延迟需求往往需自建系统。 |
| **混合架构** | 多语言分工：如 C++ 做低延迟与核心计算，Python 做研究与策略逻辑，Go 做服务与中间件。 | 兼顾开发效率与运行性能，是不少量化团队的选型方式。 |
| **算力** | 计算资源（CPU/GPU/集群）的规模与效率。 | 因子挖掘、大规模回测、机器学习训练、高频信号计算都对算力有要求；云与本地集群结合常见。 |
| **实盘 / 仿真** | **实盘**：真实资金、真实成交；**仿真**：模拟资金、模拟撮合，用于验证逻辑与系统。 | 仿真无法完全复现滑点与冲击，实盘前应有清晰的上线流程与风控参数。 |

## 五、市场与操作类（市场状态 / 日常操作）

涵盖市场环境、日常交易行为与风险管理用语，便于团队沟通与流程规范。

### 5.1 市场环境

| 概念 | 说明 | 拓展 |
|------|------|------|
| **卷** | 策略同质化、竞争加剧，超额收益被摊薄、获取难度上升。 | 需持续迭代因子与逻辑、控制容量、或转向差异化数据与策略。 |
| **水土不服** | 在某一市场或回测中有效的策略，在另一市场或实盘表现变差甚至失效。 | 可能原因：制度差异、投资者结构、流动性、数据质量、前视/幸存者偏差等。强调样本外与多市场验证。 |
| **黑天鹅** | 发生概率低、影响极大的极端事件（如股灾、政策突变、流动性危机）。 | 模型与回测难以覆盖，需靠风控（杠杆、敞口、止损）、压力测试与预案来缓释。 |
| **磨底 / 逼空** | **磨底**：价格在低位反复震荡、方向不明；**逼空**：多头强势推高价格，空头被迫平仓加剧上涨。 | 影响趋势与反转类策略的表现，需在策略设计时考虑不同市场状态。 |

### 5.2 日常操作与仓位

| 概念 | 说明 | 拓展 |
|------|------|------|
| **复盘 / 盯盘** | **复盘**：盘后回顾当日成交、盈亏与异常，分析原因；**盯盘**：盘中监控行情与策略信号、风控指标。 | 复盘用于迭代策略与规则；盯盘用于及时发现异常与执行问题。 |
| **调仓** | 根据策略信号对组合进行部分买入/卖出，调整权重或成分。 | 调仓频率与换手率影响成本与冲击；需在收益与成本间权衡。 |
| **换仓** | 将当前持仓全部平掉，再按新信号建新仓，相当于一次“全量”调仓。 | 换手大、冲击与成本高，多用于策略切换或大周期再平衡。 |
| **建仓 / 减仓** | **建仓**：逐步买入形成持仓；**减仓**：逐步卖出降低持仓。 | 可结合 TWAP/VWAP 或分批规则，平滑冲击与价格影响。 |
| **仓位 / 满仓 / 空仓** | **仓位**：持仓占可投资金的比例；**满仓**：全部资金用于持仓；**空仓**：无持仓。 | 仓位管理是风控与资金管理的核心之一，与波动率、回撤目标相关。 |
| **对冲** | 用衍生品（期货、期权等）或反向头寸，抵消组合的系统性风险（如 Beta、行业、风格）。 | 市场中性、指增控制跟踪误差都会用到对冲；需考虑基差、展期与成本。 |
| **敞口** | 未被对冲掉的风险暴露，如市值敞口、行业敞口、因子敞口。 | 敞口暴露了“赌”的方向与大小，需与产品目标与风控一致，并定期检查。 |
