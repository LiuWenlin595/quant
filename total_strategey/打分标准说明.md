# 策略综合打分标准说明

所有分数均为 **0～100 分**，缺失指标该项按 0 分参与加权。综合分 = 各子项加权和，保留一位小数。

## 一、权重

| 子项       | 权重   | 说明                                       |
|------------|--------|--------------------------------------------|
| 夏普分     | 22.5%  | 风险调整收益                               |
| 回撤分     | 18%    | 最大回撤控制                               |
| 信息比率分 | 13.5%  | 相对基准的稳定超额                         |
| 年化分     | 9%     | 策略年化收益                               |
| 胜率分     | 7.2%   | 单笔胜率（适中更优）                       |
| 盈亏比分   | 6.3%   | 盈利/亏损比                                |
| 交易次数分 | 4.5%   | 样本量（越多越可信）                       |
| 阿尔法分   | 4.5%   | 超额收益能力                               |
| 索提诺分   | 4.5%   | 下行风险调整收益                           |
| **下单时段分** | **10%** | 非 9:25～9:30 开盘下单（盘中/尾盘）得 100，9:25～9:30 下单得 0（滑点大） |

## 二、各子项计分规则（0～100）

- **夏普分**：夏普比率 ≤0→0；0.5→25；1→45；1.5→65；2→80；2.5→90；3→95；≥4→100。
- **回撤分**：最大回撤(%) 越小越好。≥60%→5；50%→15；40%→30；30%→45；25%→55；20%→65；15%→78；12%→88；10%→92；5%→97；0%→100。
- **信息比率分**：<0 按比例给 0～20；0→20；0.5→45；1→65；1.5→80；2→90；≥2.5→100。
- **年化分**：年化收益(%) <0 给低分；0→15；20%→35；50%→55；80%→70；100%→80；150%→90；≥200%→100。
- **胜率分**：0.45～0.75 区间给分更高；0.7 附近最高（92 分）；过高(>0.85)略降分。
- **盈亏比分**：0.5→20；1→45；1.5→65；2→80；2.5→90；≥3→100。
- **交易次数分**：盈利次数+亏损次数。<20→20；30→45；50→60；80→72；100→82；150→88；200→92；≥300→100。
- **阿尔法分**：<0 按比例；0→25；0.2→50；0.4→70；0.6→85；≥0.8→100。
- **索提诺分**：≤0→0；0.5→30；1→55；1.5→72；2→85；2.5→92；≥3→100。
- **下单时段分**：通过扫描策略代码中是否出现 `run_daily(..., '09:25'`～`'09:30'`（或 `"9:25"`～`"9:30"`）判断。若在 9:25～9:30 执行下单则该项 **0 分**（开盘集中成交、滑点大）；盘中或尾盘下单、或未找到对应 .py 文件则 **100 分**。只考虑盘中/尾盘下单，不考虑 9:25～9:30 下单。

## 三、CSV 新增列

- **综合分**：加权总分（一位小数）。
- **夏普分、回撤分、信息比率分、年化分、胜率分、盈亏比分、交易次数分、阿尔法分、索提诺分、下单时段分**：各子项分数（整数）。
- **打分细节**：一行文本，格式为「夏普:xx 回撤:xx … 下单时段:xx」，便于快速查看各维度。

## 四、如何复算

在项目目录执行：

```bash
python3 score_strategies.py
```

会读取 `total_strategy.csv`，按上述规则计算每行综合分与打分细节，并写回同一文件（在原有列后追加上述新列）。策略代码路径为项目下的 `Selection_Strategy/<策略名称>.py`，用于检测是否 9:30 下单。
