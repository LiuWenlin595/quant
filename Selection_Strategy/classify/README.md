# 策略按「策略主体 + 变种」分类总览

本目录对 2023–2025 年共约 400 套量化策略按**同一套策略的多个变种**进行归类：主体方法相同的策略归入同一策略族，不同实现（因子、择时、风控、标的等）视为该族的变种。

## 分类维度说明

- **策略族** = 一个子文件夹，对应一种**主体方法**（如「小市值轮动」「ETF 动量轮动」「打板/龙头」）。
- **变种** = 同一主体下的不同演进：例如小市值族内可有「+股息率」「+国九条」「+ML 选股」「微盘 400」「1/4 月空仓」等。

描述不清或无法判断主体方法的策略归入 **15_其他**，供后续人工复核或排除。

## 文件结构

每个策略族目录下：

- **README.md**：该族主体方法说明 + 变种概览 + 文件清单。
- **变种策略汇总.md**：本族**主干思想**、**变种维度归纳**、以及每个子策略的**变种要点**与**变种原因**（由脚本从各策略说明 .md 中抽取生成）。
- **2023_origin_explain / 2024_1_origin_explain / 2024_2_origin_explain / 2025_origin_explain**：说明文档（.md）副本。
- **2023_origin_py / 2024_1_origin_py / 2024_2_origin_py / 2025_origin_py**：策略代码（.py）副本。

原始文件仍保留在 `../202x_origin_explain` 与 `../202x_origin_py`，本目录为按策略族整理的副本，便于按「主体+变种」查阅与对比。

## 分类报告

- **classification_report.csv**：每行对应一个策略文件，含批次、文件名、策略族、原分类等，便于复核与调整。

## 如何重新生成

在 `Selection_Strategy` 目录下执行：

```bash
python3 classify_strategies.py
```

会覆盖 `classify/` 下各策略族子目录中的文件并重新生成各 README 与 classification_report.csv。

生成或更新各策略族下的**变种策略汇总**文档（主干思想 + 变种维度归纳 + 各子策略变种与原因）：

```bash
python3 generate_variant_summary.py
```

会为每个策略族生成或覆盖 `变种策略汇总.md`。
